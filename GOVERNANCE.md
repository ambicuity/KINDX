# KINDX Governance

KINDX operates under a **Benevolent Dictator for Life (BDFL)** model. This structure ensures decision-making remains coherent, architecturally sound, and aligned with the project's core mission: delivering enterprise-grade, on-device Knowledge Infrastructure for AI agents.

## Project Leadership

KINDX is led and maintained by **@ambicuity** (Ritesh Rana).

The BDFL is responsible for:

- Defining the strategic direction and long-term roadmap of The KINDX Specification.
- Ensuring the architectural integrity, retrieval precision, and performance of the Knowledge Infrastructure engine.
- Reviewing and approving major changes to the retrieval pipeline, protocol layer, or data schema.
- Establishing and enforcing code quality, test coverage, and supply chain security standards.
- Managing community interactions and enforcing the [Code of Conduct](./CODE_OF_CONDUCT.md).

## Decision-Making Process

The BDFL retains final authority on all matters of technical direction. KINDX is transparent about how decisions are made:

1. **Architecture Proposals:** Significant changes to the retrieval pipeline, MCP protocol, or data schema must begin as a GitHub issue using the `Architecture Proposal` template. This ensures community visibility and technical vetting before implementation effort begins.
2. **Pull Requests:** All code contributions are evaluated via Pull Request. The BDFL — and any future core maintainers — review PRs for conformance with The KINDX Specification, code quality standards, and security requirements.
3. **Discussions:** General ideas, integration questions, and community-driven proposals are addressed in GitHub Discussions.

## Pathways to Maintainership

The governance model may evolve as the project scales. Contributors who consistently demonstrate:

- **Specification Ownership** — Taking a feature from proposal through production with full test coverage and documentation.
- **Production-Grade Engineering** — Writing clean, deterministic, and secure code that meets the bar of the existing codebase.
- **Community Leadership** — Supporting other contributors, triaging issues, and improving specification clarity.

...may be invited by the BDFL to join a core maintainer team with expanded merge authority.

## Conflict Resolution

Disputes regarding technical direction or community conduct are resolved by the BDFL, with the long-term stability and mission coherence of KINDX as the guiding principle.

If you have concerns about a specific decision or the governance process itself, open a GitHub Discussion to address it transparently and constructively.
