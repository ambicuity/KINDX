# KINDX Comparison Matrix — Reference Repo Gap Analysis

## Methodology
Each dimension scored against KINDX's current state, strongest references, gap severity, and fit.

---

| Dimension | KINDX Current State | Strongest Refs | Gap Severity | Proposed Improvement | Impl Difficulty | Expected Impact | Why This Fits KINDX |
|---|---|---|---|---|---|---|---|
| **Query Planning** | Single-pass structured sub-queries (lex/vec/hyde). Profile routing (fast/balanced/max_precision). No multi-step planning. | LangGraph (graph state machines), CrewAI (multi-agent delegation) | **Low** | Add adaptive query planning that auto-selects sub-query strategy based on query characteristics. No full state-machine needed. | Low | Med | KINDX already has routing profiles. A lightweight query classifier that auto-selects lex vs vec+hyde vs all-three based on query shape improves recall without user complexity. |
| **Memory** | Scoped key-value with semantic dedup, supersession chains, tags, links table. Embedding-backed search. Solid foundation. | Zep (facts + sessions + temporal decay), Letta/MemGPT (tiered core/archival/recall), Continue (workspace context) | **Med** | Add memory TTL/decay, memory consolidation (merge old facts), and memory importance scoring. Add `memory_delete` and `memory_bulk` tools. | Med | High | Agent workflows need memory lifecycle management. TTL prevents unbounded growth. Consolidation keeps memory relevant. These are surgical additions to existing schema. |
| **Streaming** | No streaming. Query returns complete result set synchronously. MCP uses stdio/HTTP batch. | Vercel AI SDK (ReadableStream, SSE), LlamaIndex (streaming response synthesis) | **High** | Add streaming partial results via SSE for HTTP transport. Emit BM25 results immediately, then vector results, then reranked final. | Med | High | Agents waiting for full rerank pipeline (500ms-2s) get blocked. Streaming BM25 hits first gives immediate partial answers. SSE fits HTTP transport naturally. |
| **Multimodal RAG** | Markdown + PDF (via strings fallback) + DOCX adapter. 3784 docs with extractor warnings. | Unstructured (20+ formats), Haystack (converters), LlamaIndex (node parsers) | **Med** | Improve PDF extraction quality. Add structured metadata extraction. Extractor health reporting. | Med | Med | 3784 ingestion warnings indicate real quality issues. Better extraction = better retrieval. Local-first constraint means no cloud OCR. |
| **Context Compression** | None. Full chunks passed to reranker and returned to agent. | LLMLingua (token-level compression), LlamaIndex (summary index, tree summarize) | **Med** | Add result snippet compression: truncate returned snippets to most relevant sentences using extractive summarization. Add `maxTokens` parameter. | Low | High | Agents have finite context windows. Returning 10 full chunks wastes tokens. Extractive snippet compression is cheap (no LLM needed) and high-impact. |
| **Tool/Agent Hooks** | MCP tool registry with provenance. Subagent contract (typed). Tool policy enforcement. Good foundation. | AutoGPT (command registry + plugins), OpenDevin (action space), Continue (slash commands) | **Low** | Add tool execution telemetry (timing, success/failure counts). Add `tool_list` MCP tool for dynamic discovery. | Low | Med | Already well-architected. Telemetry adds observability. Dynamic tool list helps multi-agent setups. |
| **Observability** | Prometheus metrics endpoint. Structured logging. Query replay artifacts. Diagnostics/doctor. Strong. | Phoenix (OTEL spans, retrieval evals), LangSmith (trace hierarchies), Helicone (cost tracking) | **Med** | Add structured query trace spans with parent-child relationships. Emit trace events for each pipeline stage (expand → embed → retrieve → rerank). | Med | High | Query replay is good but post-hoc. Real-time trace spans enable live debugging. OTEL-compatible format enables future integration without lock-in. |
| **Policy/Security** | Multi-tenant RBAC (admin/editor/viewer). Bearer token auth. Rate limiting. Collection-scoped access. Solid. | OPA (policy-as-code), SpiceDB (relationship-based ACL) | **Low** | Add audit logging for sensitive operations. Add tenant activity summary endpoint. | Low | Med | RBAC is already production-grade. Audit logging is the missing piece for enterprise compliance. |
| **Knowledge Graph** | None. Flat document store with FTS + vector. | GraphRAG (entity extraction + community summarization), Kuzu (embedded graph DB) | **High** | Phase 1: Add document-level link graph (cross-references between docs). No full KG yet. | High | Med | Full KG is premature for local-first. But a lightweight document link graph (which doc references which) improves multi-hop retrieval. |
| **DX** | CLI with rich UX, piped output, JSON mode. MCP inspector integration. Nix flake. | Flowise (visual builder), open-webui (chat UI), Continue (IDE integration) | **Med** | Add `kindx init` wizard improvements. Add `kindx benchmark` one-liner. Add health check on first query. | Low | Med | First-run experience matters. A health check that warns about missing embeddings before first query prevents confusion. |
| **Evaluation** | Evaluation harness with BM25 quality, latency tracking, regression tests. Customer POV launch gates. Strong. | Phoenix (retrieval metrics: NDCG, MRR, precision@k), LangSmith (dataset-driven eval) | **Low** | Add NDCG@k and MRR metrics to evaluation harness. Emit per-query eval scores in replay artifacts. | Low | High | Existing harness is extensible. Adding standard IR metrics makes benchmark results comparable to literature. |
| **Packaging** | npm package, Docker, Nix flake. Workspaces for schemas/client. Python LangChain integration. | anything-llm (one-click install), khoj (Docker + pip), Continue (VS Code extension) | **Low** | Add `npx @ambicuity/kindx` zero-install runner. | Low | Med | Reduces install friction from `npm install -g` to single npx command. |
| **Enterprise Readiness** | Multi-tenant RBAC, encryption, backup/restore, sharding, diagnostics. Strong. | SpiceDB (audit trail), OPA (policy versioning) | **Low** | Add operation audit log. Add tenant usage metrics. | Low | Med | Missing piece is auditability. |
| **Local-First Ergonomics** | SQLite + WAL, on-device GGUF models, air-gapped operation, no cloud deps. Excellent. | khoj (local mode), anything-llm (local LLMs), open-webui (Ollama integration) | **Low** | Add offline readiness check. Add model size advisor based on available RAM/VRAM. | Low | Med | Already best-in-class for local-first. |
